# Pulp — Tauri Migration Plan (TBD)

Turn the Next.js web app into a native macOS app using Tauri v2.

## Why Tauri

- ~5MB binary (vs Electron's ~150MB)
- ~30MB RAM idle (vs Electron's ~300MB)
- Native WKWebView on macOS — same engine as Safari
- Rust backend for secure API key storage and HTTP calls
- No bundled Chromium, no Node.js runtime

## Strategy: Next.js → Vite + React

Next.js adds zero value in a desktop app — no SSR, no edge, no image optimization. Keeping it means fighting `output: 'export'` restrictions and carrying dead weight. The move to Vite is clean because Pulp is already fully client-side (`"use client"` on every page).

**What changes:**

| Concern | Next.js (now) | Vite + Tauri (after) |
|---|---|---|
| Routing | App Router file conventions | `react-router` (2 routes) |
| Fonts | `next/font/local`, `next/font/google` | `@fontsource` or self-hosted in `/public` |
| API routes | `/api/pulp`, `/api/draft` | Tauri HTTP plugin (direct Anthropic calls) |
| Theme | `next-themes` | Lightweight custom hook or keep `next-themes` (it works outside Next) |
| Storage | `localStorage` | `tauri-plugin-store` (JSON file on disk, no 5MB cap) |
| API key | `localStorage` | `tauri-plugin-store` (encrypted at rest) |
| Build | `next build` | `vite build` → `tauri build` |

**What stays identical:**

- All of `globals.css` (Tailwind v4, `@theme`, dark mode, type scale)
- TipTap editor (`canvas.tsx` and all extensions)
- Every component except minor import path changes
- The entire typography system

# → Window title: Pulp

### 5. Replace API routes with direct Anthropic calls

Install the Tauri HTTP plugin:

```bash
pnpm tauri add http
pnpm add @tauri-apps/plugin-http
```

Create `src/lib/anthropic.ts`:

```ts
import { fetch } from "@tauri-apps/plugin-http";
import { getApiKey } from "./api-key";
import { PULP_SYSTEM, PULP_USER, DRAFT_SYSTEM, DRAFT_USER, POLISH_SYSTEM, POLISH_USER } from "./prompts";

const API_URL = "https://api.anthropic.com/v1/messages";
const MODEL = "claude-sonnet-4-6-20250514";

async function headers() {
  const key = await getApiKey();
  return {
    "x-api-key": key!,
    "content-type": "application/json",
    "anthropic-version": "2023-06-01",
  };
}

export async function pulp(text: string, roundNumber: number, direction: string) {
  const res = await fetch(API_URL, {
    method: "POST",
    headers: await headers(),
    body: JSON.stringify({
      model: MODEL,
      max_tokens: 4096,
      temperature: 0.7,
      system: PULP_SYSTEM,
      messages: [{ role: "user", content: PULP_USER(text, roundNumber, direction) }],
    }),
  });

  if (!res.ok) throw new Error(`API error: ${res.status}`);
  const data = await res.json();
  // Parse the text content from Anthropic's response format
  const content = data.content[0].text;
  let jsonStr = content.trim();
  if (jsonStr.startsWith("```")) {
    jsonStr = jsonStr.replace(/^```(?:json)?\n?/, "").replace(/\n?```$/, "");
  }
  return JSON.parse(jsonStr);
}

export async function draft(text: string, direction: string, mode: "draft" | "polish") {
  const isPolish = mode === "polish";
  const res = await fetch(API_URL, {
    method: "POST",
    headers: await headers(),
    body: JSON.stringify({
      model: MODEL,
      max_tokens: 4096,
      temperature: isPolish ? 0.3 : 0.6,
      stream: true,
      system: isPolish ? POLISH_SYSTEM : DRAFT_SYSTEM,
      messages: [{ role: "user", content: isPolish ? POLISH_USER(text, direction) : DRAFT_USER(text, direction) }],
    }),
  });

  if (!res.ok) throw new Error(`API error: ${res.status}`);
  return res.body; // ReadableStream — consume in the component
}
```

**Note on streaming:** Tauri's HTTP plugin supports `ReadableStream` on the response body. The existing streaming logic in `Write.tsx` (the `reader.read()` loop) works with minimal changes — just swap the fetch call.

Configure allowed domains in `src-tauri/capabilities/default.json`:

```json
{
  "permissions": [
    {
      "identifier": "http:default",
      "allow": [{ "url": "https://api.anthropic.com/*" }]
    }
  ]
}
```

### 6. Replace localStorage with tauri-plugin-store

```bash
pnpm tauri add store
pnpm add @tauri-apps/plugin-store
```

Update `src/lib/store.ts` — replace `localStorage.getItem/setItem` with:

```ts
import { LazyStore } from "@tauri-apps/plugin-store";

const store = new LazyStore("pulp-sessions.json", { autoSave: true });

async function loadSessions(): Promise<Session[]> {
  const raw = await store.get<Session[]>("sessions");
  return raw ?? [];
}

async function saveSessions(sessions: Session[]) {
  await store.set("sessions", sessions);
}
```

The `useSessions` hook becomes async-aware (load sessions in `useEffect`, save with `await`). The shape of the hook's return value stays the same — components don't change.

Update `src/lib/api-key.ts` similarly:

```ts
import { LazyStore } from "@tauri-apps/plugin-store";

const store = new LazyStore("pulp-settings.json", { autoSave: true });

export async function getApiKey(): Promise<string | null> {
  return (await store.get<string>("apiKey")) ?? null;
}

export async function setApiKey(key: string) {
  await store.set("apiKey", key);
}

export async function removeApiKey() {
  await store.delete("apiKey");
}
```

### 7. Replace `next-themes`

Two options:

**Option A — keep `next-themes`.** It works outside Next.js (it's just a React context + DOM attribute setter). Only import changes.

**Option B — tiny custom hook.** Since Pulp only needs light/dark/system:

```ts
// src/lib/use-theme.ts
import { useState, useEffect, createContext, useContext } from "react";

type Theme = "light" | "dark" | "system";
const ThemeContext = createContext<{ theme: Theme; setTheme: (t: Theme) => void }>({
  theme: "system", setTheme: () => {},
});

export function ThemeProvider({ children }: { children: React.ReactNode }) {
  const [theme, setTheme] = useState<Theme>("system");

  useEffect(() => {
    const resolved = theme === "system"
      ? window.matchMedia("(prefers-color-scheme: dark)").matches ? "dark" : "light"
      : theme;
    document.documentElement.setAttribute("data-theme", resolved);
  }, [theme]);

  return <ThemeContext value={{ theme, setTheme }}>{children}</ThemeContext>;
}

export const useTheme = () => useContext(ThemeContext);
```

Option B is better — removes a dependency and it's 20 lines. In a native app, system theme detection via `prefers-color-scheme` still works in WKWebView.

### 8. Replace Next.js navigation

In `Home.tsx` and `Write.tsx`, replace:

```ts
// Before
import { useRouter } from "next/navigation";

## Known risks and mitigations

### TipTap in WKWebView
There are reported edge cases with TipTap's heading and link extensions in WebKit webviews. Setting `minimumSystemVersion: "13.0"` (Safari 16.4+) avoids the older WebKit bugs. Test thoroughly — if you hit issues, pin your TipTap version.

### Streaming responses
Tauri's HTTP plugin supports streaming via `ReadableStream`. The existing `reader.read()` loop in the write page should work directly. If it doesn't, fallback is a Rust command that streams via Tauri events.

### Data migration
Users of the web version have data in `localStorage`. If you want to support migration, you could:
1. Ship a one-time "import from browser" feature
2. Or accept that the native app starts fresh (likely fine for a new launch)

### Code signing for distribution
Without signing + notarization, macOS will block the app with Gatekeeper. You need:
- Apple Developer account ($99/year)
- `Developer ID Application` certificate
- Notarization via `xcrun notarytool` (Tauri handles this automatically when env vars are set)

For personal/testing use, ad-hoc signing (`"-"`) works fine.
